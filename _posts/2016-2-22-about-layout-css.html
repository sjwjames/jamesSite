不知不觉到上海来已经三个月了，在心动满打满算也三个月了。这三个月里面主要还是以前端工作为主，当然也参加了一些后端的工作，包括后面将要参加的项目。 刚去的时候，其实我的前端知识仅限于大三上学的web课程和课下看的一本薄薄的html5的书。细细回忆一下，实习中大概大大小小做了有近10个页面。主要踩的坑是在布局、响应式以及对于UI设计图的把握。以下是我的一些总结，有错误的地方欢迎指正。

由于前端的工作是将设计师的设计图以代码的形式呈现出来，并且要做到pixel perfect，所以工作的起点是拿到设计图和切好的图片，而第一个接触的人（如果是 单兵作战，产品锦鲤不算）就是UI设计师。我们通常首先需要和设计师讨论他们的设计意图，包括页面将要在哪些终端呈现，是以手机版+PC的方式还是以响应式的方式， 背景图片的整体宽度和主体宽度是否合理，主体宽度是否满足最小屏幕的需求，在响应式设计中，布局需要如何动态改变，如果需要支持（一般都需要）retina屏幕，或者 更加高清的屏幕时候，是否有双倍大小的切图以保证高清屏幕上图片的清晰。在这个过程中，需要不断沟通，由于设计师一般都不懂前端技术，一定要推动设计师做出 正确的设计和有利于开发的设计。

假设pc版的图是1920px宽的，那么它的主体部分需要在居中并且宽度大致为1000px(或者一个合适的值，这个值并不固定)以满足小屏幕的宽度也能看到主要的内容，当然越宽的屏幕看到的 东西越丰富，虽然超过主体内容范围的都是一些次要的内容。如果设计师给了两张图，一张是超过主体部分的次要背景，class="bg"，一张是主体部分,class="main-content"，
.bg{ width:100%; background-image:url("bg.jpg"); background-repeat:no-repeat; background-size:auto; background-position:center 0; } .main-content{ width:1000px;/*不一定是1000px根据需求来变化*/ background-image:url("main-content.jpg"); background-repeat:no-repeat; background-size:auto; background-postion:center 0; margin:auto; }
这样就能使主体一直处于屏幕中间。而如果设计师只给了一张图，只需要把main-content的width改成100%,设置min-width为1000px;

有的时候需要在一张背景图上定位，而又要做到无论终端宽度怎么变化位置都不会有变化，就要让背景以相对定位的方式来定位，设定background-size为100% ，并使用padding-top百分比的形式来指定其高度，其上的元素则使用绝对定位并使用百分比做单位来进行定位，此时，背景保证会撑满屏幕宽度，而高度会随着屏幕宽度变化而变化， 因为padding-top会根据父元素的宽度来进行计算，因此背景高度依赖于屏幕宽度，子元素使用绝对定位来确定自己的位置就相当简单。

其次遇到过的比较蛋疼的情况就是居中的问题了，这个我就留到下篇博客细致记录一下。
